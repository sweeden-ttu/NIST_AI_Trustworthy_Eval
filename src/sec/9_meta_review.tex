\section{Meta-Review and Integration}
\label{sec:meta-review}

The fourteen prompts collectively stress-test a single target model across \textbf{NIST AI RMF} trustworthy characteristics (valid/reliable, safe, secure and resilient, explainable, privacy-enhanced, fair, accountable and transparent). The narrative in \texttt{experiment.tex} (built via \texttt{src/main.tex}) ties each item to a research question, hypothesis, and rubric label; this meta-review checks that integration is coherent rather than a disconnected list of refusals or boilerplate.

\strength{Traceability is explicit: prompts are versioned in \texttt{scripts/nist\_quiz\_prompts.py}, responses in \texttt{output/results/nist\_eval\_latest.json}, and human scores in \texttt{output/results/nist\_quiz\_scores.json}.}

\subsection*{Reproducibility (IEEE build)}
Regenerate evaluation artifacts from the repository root:
\begin{itemize}
  \item \textbf{Prompt inventory (no API key):} \texttt{python scripts/verify\_nist\_prompt\_inventory.py}
  \item \textbf{Live run:} set \texttt{OPENAI\_API\_KEY}, optional \texttt{OPENAI\_BASE\_URL}, \texttt{NIST\_EVAL\_MODEL}; then \texttt{python scripts/run\_nist\_llm\_evaluation.py}
  \item \textbf{Orchestrated refresh:} \texttt{python scripts/run\_coursework\_outputs.py} (reads \texttt{test\_cases/CONFIG}; runs the live NIST driver unless \texttt{--skip-nist-eval})
  \item \textbf{Rubric table:} edit \texttt{output/results/nist\_quiz\_scores.json}, then \texttt{python scripts/emit\_nist\_rubric\_table.py}
\end{itemize}

\noindent\textbf{Note:} Legacy applied-cryptography coursework (OpenSSL, toy AES figures) is \textbf{not} part of the Quiz~\#3 reproducibility path. If you still maintain that track, document it separately and do not confuse it with the NIST JSON pipeline above.

For LangGraph/\texttt{rd-agent-mcp}, set \texttt{LM\_STUDIO\_BASE\_URL} and use \texttt{RD\_AGENT\_MCP\_EXEC\_AGENTS=true} only in trusted environments when spawning real \texttt{rd-agent} / \texttt{adk-ralph} subprocesses.
