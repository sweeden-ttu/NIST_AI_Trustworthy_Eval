\subsection{Question 1: Toy AES Hamming traces}
We compare all-zero AES-128 plaintext $P$ against $P'$ differing by a single bit in byte~0 (value \texttt{0x01}), with an all-zero key unless noted. Figure~\ref{fig:q1-base} shows bit Hamming distance after each traced step for the standard round function. Figure~\ref{fig:q1-sr-off} disables ShiftRows (\texttt{TraceOpts(disable\_shift\_rows=True)}). Figure~\ref{fig:q1-mnew} uses custom MixColumns matrix \texttt{M\_NEW} from \texttt{toy\_aes128\_trace.py}. JSON summaries, including Part~D key-flip rows, are in \texttt{output/results/q1-summary.json}.

\subsection{Question 2: ECB vs.\ CBC on BMP data}
OpenSSL \texttt{enc} applies AES-128-ECB and AES-128-CBC (\texttt{-nosalt}) to \texttt{assets/Secret.bmp} (a minimal gradient BMP is generated if the file is absent). Figure~\ref{fig:q2-noise} visualizes ciphertext \emph{bodies} as grayscale grids; Figure~\ref{fig:q2-fix} prepends the 54-byte plaintext BMP header to ciphertext so ECB structural leakage remains visible while CBC appears more uniform at a glance \citep{dworkin2007sp800-38a}. Key and IV hex for replay are stored in \texttt{output/results/q2/openssl\_run.json} (handle per your data policy).

\subsection{Question 3: TLS negotiation snapshot}
\texttt{openssl s\_client} connects to nine public HTTPS hosts and records negotiated protocol and cipher strings. Figure~\ref{fig:q3-table} tabulates the sample; Figure~\ref{fig:q3-bar} counts cipher reuse. These plots are \textbf{not} a substitute for full Wireshark ClientHello classification required by typical coursework; they provide a reproducible PDF stand-in. Raw capture exports can be placed under \texttt{output/diagrams/q3-wireshark/}.
